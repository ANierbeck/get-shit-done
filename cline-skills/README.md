# Cline Skills for Get Shit Done

This package provides Cline-compatible skills that integrate with the Get Shit Done (GSD) workflow. These skills follow the official Cline skills format while leveraging GSD principles and infrastructure.

## About Cline Skills

Cline skills are modular capabilities that extend Cline's functionality. Each skill is a self-contained directory with:

```
skill-name/
├── SKILL.md          # Required: main instructions with YAML frontmatter
├── docs/             # Optional: additional documentation
│   └── advanced.md
└── scripts/          # Optional: utility scripts
    └── helper.sh
```

## Available Skills

### 🔍 Code Review

**Skill**: `code-review`

**Description**: Perform comprehensive code review with automated analysis, quality metrics, and actionable recommendations.

**Use When**:
- You need to analyze code quality
- You want to identify security vulnerabilities
- You need to detect performance issues
- You want to generate improvement plans

**Features**:
- Automated static analysis (ESLint, Pylint, etc.)
- Complexity analysis with Lizard
- Duplicate code detection
- Security vulnerability scanning
- Quality metrics calculation
- GSD-compatible fix plan generation

### ⚡ Performance Audit

**Skill**: `performance-audit`

**Description**: Conduct performance analysis to identify bottlenecks, measure metrics, and provide optimization recommendations.

**Use When**:
- You need to analyze application performance
- You want to establish performance baselines
- You need to optimize critical code paths
- You want to identify scaling bottlenecks

**Features**:
- System baseline metrics capture
- Code complexity analysis
- Performance pattern detection
- Dependency impact analysis
- Bottleneck identification
- Optimization recommendations
- GSD-compatible optimization plans

## Installation

### Prerequisites
- Node.js 16.7.0 or higher
- Get Shit Done initialized in your project (`/gsd:new-project`)

### Install Skills

```bash
# Install to default Cline skills directory
node cline-skills/install.js

# Show help
node cline-skills/install.js --help
```

### Custom Installation Location

Set `CLINE_SKILLS_DIR` environment variable to specify custom installation location:

```bash
export CLINE_SKILLS_DIR=/custom/path/to/skills
node cline-skills/install.js
```

## Skill Structure

Each skill follows the official Cline format:

### SKILL.md Format

```markdown
---
name: skill-name          # Must match directory name
description: Brief description of what this skill does and when to use it.
---

# Skill Title

## Steps

1. **Step 1**: Description of what this step does
2. **Step 2**: Description of what this step does
3. **Step 3**: Description of what this step does

## Advanced Usage

For advanced configuration, see [advanced.md](docs/advanced.md).

## Requirements

- Any prerequisites or dependencies

## Output

- Files and artifacts generated by this skill
```

## Integration with GSD

These skills are designed to work seamlessly with Get Shit Done:

1. **State Management**: Skills update `.planning/STATE.md` with execution results
2. **Context Engineering**: Skills reference GSD principles and workflows
3. **Task Creation**: Skills can generate GSD-compatible execution plans
4. **Progress Tracking**: Skills maintain history and context

## Creating New Skills

To create new Cline skills that integrate with GSD:

1. **Follow the Format**: Use the official Cline skills structure
2. **Integrate with GSD**: Reference GSD principles and state management
3. **Document Thoroughly**: Provide clear descriptions and usage guidance
4. **Test Compatibility**: Ensure skills work with GSD workflow


## Usage Examples

### Running a Code Review

```markdown
User: Can you review the codebase for quality issues?

Cline: [Activates code-review skill]
       [Performs automated analysis]
       [Generates comprehensive report]
       [Presents findings and recommendations]

User: Please create a fix plan for the critical issues.

Cline: [Generates GSD-compatible execution plan]
       [Creates tasks for each critical issue]
       [Provides verification criteria]
```

### Running a Performance Audit

```markdown
User: The application feels slow. Can you analyze performance?

Cline: [Activates performance-audit skill]
       [Captures baseline metrics]
       [Analyzes code complexity]
       [Identifies bottlenecks]
       [Generates optimization report]

User: Let's implement the high-impact optimizations.

Cline: [Creates GSD optimization plan]
       [Generates tasks with before/after metrics]
       [Provides implementation guidance]
```

## Development

### Adding New Skills

1. Create skill directory with proper structure
2. Write SKILL.md with clear YAML frontmatter
3. Add documentation in docs/
4. Add utility scripts in scripts/ if needed
5. Test the skill thoroughly
6. Add to installation script

### Testing Skills

```bash
# Test skill structure
node cline-skills/install.js --help

# Verify skill files
find cline-skills -name "SKILL.md" -exec echo "Found: {}" \;
```
